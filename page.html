Interactive Guide to LLM Capabilities

This guide provides a series of hands-on activities to explore the features of large language models using NotebookLM. Please select an activity from the navigation to begin. Each activity's "Objective" explains its purpose and what you'll learn.

1. Qualitative Analysis for Research

Objective

Observe how NotebookLM can assist in identifying themes across multiple documents and connect them to source material.

Instructions

  • Upload a series of documents (e.g., transcripts, survey responses, or articles) into a NotebookLM notebook. The example uses 64 YouTube videos.
  • In the Chat for this Notebook, use the following prompt: "Identify key themes in these documents. A theme should be drawn from at least 3 pieces of supporting evidence from the sources, and supporting evidence must be cited."

Reflection

  • Review the themes identified by NotebookLM. How well does it connect the themes to the source documents?
  • Consider what context might be missing from the analysis.
  • Try adding or removing documents to see how the identified themes change.
  • Evaluate the accuracy of the citations provided by NotebookLM.

2. Interpret Handwritten Notes

Objective

Observe NotebookLM's ability to process handwritten notes and images, and to test its accuracy in extracting specific information.

Instructions

  • Upload an image of a handwritten document to a NotebookLM notebook. The example uses a handwritten filing to the US Supreme Court.
  • In the Chat for this Notebook, ask a specific question about the content of the document. For example: "What is the last full sentence on page 2 of the source document? By last full sentence I mean the last sentence that begins and ends on page 2."
  • Note that NotebookLM may sometimes provide an incorrect answer (hallucinate) or state it cannot answer the question.

Reflection

  • Consider the technical limitations for reading handwritten documents.
  • Try asking about visual elements, for example: "Where is red ink on this document?"
  • If NotebookLM provides an answer, critically evaluate its accuracy by comparing it to the source document. If it cannot answer, consider why that might be.
  • Think about how the clarity of handwriting might affect NotebookLM's performance.

3. Draft Reports and Documentation

Objective

Observe how NotebookLM can synthesize information from multiple sources to generate a comparative report.

Instructions

  • Upload at least two documents that you want to compare (e.g., employee handbooks from different organizations). The example uses employee handbooks from Amherst College and Hampshire College.
  • In the Chat for this Notebook, use the following prompt: "Generate a report which identifies the similarities and differences between [Document 1 name] and [Document 2 name]."

Reflection

  • Review the generated report. How accurately does it identify similarities and differences based on the provided sources?
  • Consider how different prompts might steer the generated output. For example, how might asking for "key differences" versus a general "comparison" change the result?
  • Examine the structure and coherence of the report generated by NotebookLM.

4. Read Technical Documentation

Objective

Observe how NotebookLM can be used to quickly find relevant information within lengthy technical documentation.

Instructions

  • Upload a piece of technical documentation (e.g., a user manual) to a NotebookLM notebook. The example uses a 122-page user manual for a ThinkPad.
  • In the Chat for this Notebook, ask a specific question about the product's features or functionalities based on the manual. For example: "I have a smart card with a slit. Is this supported?"

Reflection

  • Assess how effectively NotebookLM draws from the provided source to answer your question.
  • Consider whether the outputs are complete. Does NotebookLM provide all relevant information, or does it miss important details or context from the manual?
  • Think about the types of questions for which this feature would be most useful (e.g., troubleshooting, feature compatibility, specific instructions).

5. Multimodal Learning

Objective

Observe how NotebookLM can process various information modalities and generate outputs that support multimodal learning.

Instructions

  • Upload source materials in different formats if available, such as PDFs containing images, or provide links to YouTube videos. The example mentions using lecture slides.
  • Experiment with generating different types of outputs. For instance, after uploading lecture slides, go to the "Studio" tab and create an "Audio Overview."
  • If you have documents with diagrams or structured information, try asking NotebookLM to discuss a concept in the context of another. For example: "Discuss what these sources say about Notation (vectors), in the larger context of Properties of Preferences (Assumptions)."

Reflection

  • Consider how different output formats (audio, visual like mind maps, textual) might cater to different learning preferences.
  • Critically evaluate the usefulness and accuracy of the generated multimodal outputs.
  • Remember the "Critical Eye" advice: Always ask for permission to use an instructor's materials as sources for NotebookLM if they aren't publicly available. How does this ethical consideration apply to other types of source material?
